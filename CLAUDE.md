# MD Viewer

A macOS markdown viewer built on Tauri 2 (Rust backend, vanilla JS frontend).
Renders with `marked` v17, edits with CodeMirror 6, ships as a `.app` + `.dmg`
plus a sandboxed Quick Look preview extension.

## Layout

- `src/` — frontend: `index.html`, `main.js`, `styles.css`, vendored
  `marked.esm.js` and `codemirror.bundle.js`. No runtime bundler; ships as-is.
- `src-tauri/` — Rust backend, `Cargo.toml`, `tauri.conf.json`. The macOS
  `Info.plist` is generated by Tauri at build time from `tauri.conf.json` plus
  `bundle.macOS.*` overrides; arbitrary plist keys (e.g. `UTExportedTypeDeclarations`)
  are patched in post-build.
- `quicklook/` — sandboxed macOS Quick Look extension. One Swift file built
  directly with `swiftc` (no Xcode project) into a `.appex` bundle that is
  embedded into `md-viewer.app/Contents/PlugIns/` at install time. Reuses
  `src/marked.esm.js` and `src/styles.css` so there is one rendering source
  of truth.
- `scripts/install-quicklook.sh` — builds the `.appex`, embeds it into a
  given `md-viewer.app`, patches the host plist, re-signs, refreshes Launch
  Services, and unregisters stale duplicate registrations.
- `.github/workflows/release.yml` — tag-triggered. Builds Tauri app for
  macOS arm64 + x86_64, signs with the Apple Development cert imported from
  the `MACOS_CERT_P12_BASE64` / `MACOS_CERT_PASSWORD` / `MACOS_CODESIGN_IDENTITY`
  repository secrets, runs the Quick Look install script, repackages the
  DMG with `hdiutil`, and uploads. Linux + Windows jobs are
  `continue-on-error` and untested.
- `cm-entry.js` + `npm run build:cm` — esbuild the CodeMirror bundle when
  `cm-entry.js` or its deps change. Bundle is committed.

## Common commands

```sh
npm install                 # once
npm run tauri dev           # dev build, hot reload
npm run tauri build         # production .app + .dmg
npm run build:cm            # rebuild CodeMirror bundle (only after editing cm-entry.js)

# Build & embed the Quick Look extension into a built .app:
CODESIGN_IDENTITY="Apple Development: you@example.com (TEAMID)" \
  ./scripts/install-quicklook.sh /Applications/md-viewer.app
```

## Releasing

Bump version in **all five** of: `package.json`, `package-lock.json`,
`src-tauri/Cargo.toml`, `src-tauri/Cargo.lock`, `src-tauri/tauri.conf.json`
(use `npm install --package-lock-only` and `cargo check --offline` to update
the lock files). Add a `[X.Y.Z] — YYYY-MM-DD` section to `CHANGELOG.md` plus
a `[X.Y.Z]: compare-url` link at the bottom. Tag `vX.Y.Z` and push — CI
builds and uploads the bundles.

## Things to read before editing

- `DEVELOPMENT.md` — every cross-platform CI gotcha and every Quick Look
  extension gotcha is documented there with the symptom, the diagnosis, and
  the fix. Read the relevant section before re-touching anything macOS- or
  CI-specific.
- The Quick Look extension in particular has a long list of non-obvious
  failure modes (sandbox entitlements for WebKit, `NSExtensionMain` linker
  flag, `--deep` re-signing dropping entitlements, ES modules and `file://`
  origin, multiple `.app` copies confusing PlugInKit). Don't redo the
  diagnosis — `DEVELOPMENT.md` has it.

## Conventions

- Frontend uses no build step at runtime; everything in `src/` ships verbatim.
- No comments on the WHAT — let identifiers explain. Comments are reserved
  for the WHY of non-obvious choices.
- Tauri config is the source of truth for app metadata and file associations.
  Anything macOS-specific that Tauri doesn't expose (UTI exports, code
  signing identity, extension embedding) is handled by post-build scripts in
  `scripts/`.
